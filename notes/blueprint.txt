1. VISI & MODEL BISNIS
================================================================================
Rabiku.id adalah platform jasa pembuatan website wedding dengan model sewa 
berlangganan. Customer menyewa website untuk durasi tertentu (2-3 bulan) 
sesuai masa pernikahan mereka.

Model Bisnis:
- Customer (Yusuf) menghubungi via WhatsApp
- Pembayaran via QRIS (manual)
- Super Admin (pemilik) membuatkan akun setelah pembayaran
- Customer mendapat akses dashboard untuk mengedit konten
- Website aktif sesuai durasi sewa, setelah expired akan dinonaktifkan

Nama Repo GitHub:
- rbk2026 (RaBiKu 2026)
- Alasan: Singkatan dari RaBiKu, aman dari public, ga ketauan project wedding

7. SISTEM TEMA & DESAIN
================================================================================

Konsep Tema:
- 5 tema berdasarkan WARNA: Coklat, Biru, Pink, Hijau, Ungu
- Setiap warna memiliki beberapa GAYA: Minimalis, Mewah, Rustic, Modern, Romantic
- Total kombinasi: 5 warna × 3-5 gaya = 15-25 tema
- Super Admin menentukan tema untuk setiap customer saat membuat akun
- Customer TIDAK bisa mengubah tema sendiri

Variasi Desain per Section:
- Hero: 3-5 layout berbeda
- Gallery: 3 layout (grid, masonry, carousel)
- Thanks: 3-5 variasi ucapan
- Total kombinasi desain: 5 warna × 3 gaya × 3 layout = 45+ desain

Pengelolaan Tema:
- Fase 1: Hardcode 5-10 template dasar (mulai jualan)
- Fase 2: Tambah variasi layout (setelah ada 20+ customer)
- Fase 3: Super Admin dashboard untuk kelola tema (opsional, jika scale)


8. SISTEM LAGU/MUSIK
================================================================================

Fitur Audio:
- Floating music player di semua halaman website wedding
- Autoplay (dengan interaksi user pertama kali)
- Loop (repeat)
- Customer bisa pilih lagu dari daftar yang disediakan
- Lagu bisa dibedakan per tema (misal: tema coklat → lagu klasik/religi)

Opsi Implementasi:
- Fase 1: Customer pilih dari daftar 10-15 lagu (hardcode di database)
- Fase 2: Customer upload MP3 sendiri (fitur premium, extra charge)

Sumber Lagu:
- Lagu bebas royalti (Artlist, SoundCloud, dll)
- Atau embed Spotify/YouTube (tidak perlu storage sendiri)


9. TECH STACK
================================================================================

FRONTEND:
─────────────────────────────────────────────────────────
Framework      : Next.js 14+ (App Router)
Styling        : Tailwind CSS
UI Components  : Custom (tidak pakai library UI)
State Management: React Context / Zustand (jika diperlukan)
Animation      : Framer Motion / AOS (scroll animation)
Form           : React Hook Form
Toast/Notif    : React Hot Toast / Sonner

BACKEND:
─────────────────────────────────────────────────────────
Runtime        : Node.js (Next.js API Routes)
ORM            : Prisma
Database       : Neon (PostgreSQL) - Serverless
Authentication : NextAuth.js / JWT + bcrypt
Validation     : Zod

STORAGE (File/Asset):
─────────────────────────────────────────────────────────
Images         : Cloudinary (upload & optimization)
Music/Audio    : Cloudinary / Firebase Storage
Atau Embed     : Spotify / YouTube

DEPLOYMENT & HOSTING:
─────────────────────────────────────────────────────────
Hosting        : Vercel (serverless)
Database       : Neon (PostgreSQL) - terintegrasi dengan Vercel
Domain         : rabiku.id (custom domain ke Vercel)
SSL            : Otomatis dari Vercel

DEV TOOLS:
─────────────────────────────────────────────────────────
Package Manager: npm / yarn
Version Control: Git + GitHub (repo: rbk2026)
Linting        : ESLint
Formatting     : Prettier
Type Checking  : TypeScript (strict mode)

- id (PK, cuid)

- groomName

- brideParents (JSON: ayah, ibu)
- weddingDate (DateTime)
- weddingTime (String)
- venueName
- venueAddress
- loveStory (Text)
- coverImage (URL)
- galleryImages (JSON Array of URLs)
- customMessage (Text)

- // RANGKAIAN ACARA (JSON Array)
- eventSchedule (JSON: [{time, title, description}])

- // KADO / REKENING (JSON Array)
- bankAccounts (JSON: [{bankName, accountNumber, accountHolder}])
- donationLink (String)

- // THEMA (ditentukan Super Admin)
- theme (Enum: COKLAT, BIRU, PINK, HIJAU, UNGU)
- themeStyle (Enum: MINIMALIS, MEWAH, RUSTIC, MODERN, ROMANTIC)

- songId (FK ke tabel Song)
- customSongUrl (String, opsional untuk custom upload)

- // STATUS
- isActive (Boolean)
- expiredDate (DateTime)
- views (Int, counter)
- createdAt
- updatedAt

TABEL SONG (Daftar lagu default):
─────────────────────────────────────────────────────────
- id (PK, cuid)
- title
- artist

TABEL RSVP (Buku Tamu):

TABEL TESTIMONIAL (Untuk landing page):
─────────────────────────────────────────────────────────
- id (PK, cuid)
- weddingSiteId (FK ke WeddingSite)

- customerPhoto (opsional)

15. PERTIMBANGAN TEKNIS PENTING
================================================================================

1. PERFORMANCE:
   - Gunakan Next.js Image component untuk optimasi gambar
   - Lazy loading untuk section di bawah fold
   - Cache database query dengan Prisma
   - Gunakan CDN Cloudinary untuk semua asset

2. SEO:
   - Meta tags dinamis per website wedding
   - Open Graph untuk sharing ke WhatsApp/IG
   - Sitemap otomatis

3. KEAMANAN:
   - Hash password dengan bcrypt
   - Validasi input dengan Zod
   - Middleware untuk proteksi route
   - Rate limiting untuk API
   - CORS configuration
   - Environment variables untuk semua secret

4. SCALABILITY:
   - Database indexing (slug, email)
   - Pagination untuk daftar user & RSVP
   - Optimasi query (jangan n+1 problem)
   - Serverless siap untuk scale otomatis (Vercel)

5. USER EXPERIENCE:
   - Loading state di semua async operation
   - Toast notification untuk feedback
   - Skeleton loading untuk gambar
   - Drag & drop upload gambar
   - Auto-save saat edit konten
